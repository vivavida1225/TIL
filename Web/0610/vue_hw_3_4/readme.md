# v-model을 안 쓰려면?

---
한글과 같은 IME(입력기) 조합형 문자들은 v-model을 쓰면 한 글자(음절)가 완전히 완성되기 전까지는 데이터가 실시간으로 반영되지 않는 답답한 현상이 있죠.

이 문제를 해결하기 위해 v-model 대신 v-bind:value와 @input을 조합하는 방식은 한국 웹 프론트엔드 개발자라면 반드시 알아야 하는 필수 치트키입니다!
---
💡 한글 입력 최적화 핵심 원리
v-model은 내부적으로 :value와 @input을 자동으로 묶어주는 편리한 도구이지만, 내부적으로 "글자 조합 중(Composition)"일 때는 데이터 업데이트를 잠시 멈추도록 설계되어 있습니다. 영어와 달리 한글은 초성·중성·종성이 모여 하나의 글자가 되기 때문입니다.

따라서 조합 중인 자음·모음(예: 'ㄱ', '가')까지 실시간으로 낚아채려면, 이 수동 제어 고삐를 직접 쥐어야 합니다.

v-bind:value (:value): 스토어나 스크립트의 데이터를 인풋창에 보여줍니다.

v-on:input (@input): 키보드가 눌릴 때마다 조합 여부와 상관없이 무조건 실시간으로 값을 빼앗아 데이터를 갱신합니다.
---

```html
<!-- 둘은 같은 역할을 하지만, 후자는 한글에 즉각적으로 반응한다! -->

<input type="text" v-model="object.text">

<input type="text" :value="object.text" @input="object.text = $event.target.value">
```
